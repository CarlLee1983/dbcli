<objective>
Render the security notification in the table and CSV output formats of QueryResultFormatter when blacklisted columns have been omitted from results.

Purpose: The security notification string is correctly stored in result.metadata.securityNotification by QueryExecutor when columns are filtered. In JSON format this is already visible. In table format (the default) it is silently dropped, meaning a user receiving table output has no indication that their results are incomplete. This is a security transparency gap.

Output: One modified file. formatTable() and formatEmptyTable() append the securityNotification to the footer when present. formatCSV() appends it as a comment line. One new test file verifying the rendering.
</objective>
