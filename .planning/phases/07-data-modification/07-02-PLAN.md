Enhance existing DataExecutor class with UPDATE-specific SQL building:

**Add buildUpdateSql method to DataExecutor class:**
```typescript
private buildUpdateSql(
  tableName: string,
  data: Record<string, any>,
  whereClause: string,
  schema: TableSchema
): { sql: string, params: any[] }
```

**Implementation steps:**

1. **Validate WHERE clause is provided:**
   - Throw if whereClause is undefined or empty string
   - Error: "UPDATE requires --where clause"

2. **Validate all SET fields exist in schema:**
   - Check each key in data exists in table schema columns
   - Throw error: "Column '{column}' not found in table '{table}'"

3. **Parse WHERE clause into parameterized form:**
   - Extract condition from whereClause (e.g., "id=1" or "id > 5")
   - Replace literal values with placeholders (?, $N)
   - Build params array: SET params first, then WHERE params
   - Example: "id=1 AND status='active'" → "id=? AND status=?" with params extracted

4. **Build SET clause with placeholders:**
   - For PostgreSQL: `UPDATE users SET name=$1, email=$2, ...`
   - For MySQL: `UPDATE users SET name=?, email=?, ...`
   - Use consistent column ordering

5. **Combine SET and WHERE with proper parameter ordering:**
   - SET parameter indices come first ($1, $2, ... for PostgreSQL)
   - WHERE parameter indices follow ($3, $4, ...)
   - Example PostgreSQL: `UPDATE users SET name=$1, email=$2 WHERE id=$3`
   - Example PostgreSQL params: ["Alice", "a@b.com", 1]

6. **Return { sql, params } for adapter.execute()**

**Error Handling:**
- Column not found: throw descriptive error before execution
- WHERE clause missing: throw before attempting SQL generation

```typescript
  tableName: string,
  data: Record<string, any>,
  whereClause: string,
  schema: TableSchema,
  options?: DataExecutionOptions
): Promise<DataExecutionResult>
```

**Implementation steps:**

1. **Validate WHERE clause is provided:**
   - Reject if whereClause is undefined or empty string
   - Error: "UPDATE requires --where clause (e.g., --where \"id=1\")"

2. **Enforce permission for UPDATE:**
   - Call `enforcePermission('UPDATE', this.permission)`
   - Catch PermissionError with message (per D-05b): "Permission denied: Query-only mode allows SELECT only. Use Read-Write or Admin mode for UPDATE."

3. **Build UPDATE SQL via buildUpdateSql()**

4. **Handle --dry-run mode (per D-03c):**
   - If `options?.dryRun === true`: return result with rows_affected=0, don't execute

5. **Handle --force mode (per D-03b):**
   - If `options?.force === true`: skip confirmation prompt, execute immediately

6. **Handle confirmation flow (per D-03a):**
   - If neither dry-run nor force: call `promptConfirmation(sql)`
   - Display SQL, wait for y/n
   - If 'n': return early without executing

7. **Execute via adapter:**
   - Call `this.adapter.execute(sql, params)`
   - Capture affected row count

8. **Handle errors:**
   - Catch database errors and augment with generated SQL (per D-05a)

9. **Return DataExecutionResult with status, operation, rows_affected, timestamp**

**Error Cases:**
- WHERE clause missing → validation error
- Permission denied → PermissionError re-thrown
- Column not found → caught by buildUpdateSql
- Database error → augmented with SQL
