<objective>
Wire BlacklistManager and BlacklistValidator into the four CLI execution commands so that blacklist rules configured in .dbcli take runtime effect.

Purpose: The infrastructure (BlacklistManager, BlacklistValidator, executor integration) was built in Plan 01 but the CLI layer never constructs or passes the validator. This means a blacklisted table is silently accessible from the command line — a critical security gap.

Output: Four modified command files. Each reads config.blacklist, constructs BlacklistManager + BlacklistValidator, and passes the validator to the executor. ~3 lines of change per file.
</objective>
