2. **Error categorization logic** (in order, first match wins):

   **ECONNREFUSED** - Server not running or port not listening
   - Trigger: errCode === 'ECONNREFUSED' OR message includes 'refused'
   - Message: `無法連接至 ${options.host}:${options.port} — 伺服器未運行或未監聽該埠號`
   - Hints:
     - `確認 ${system} 服務已啟動: ${system === 'postgresql' ? 'systemctl status postgresql' : 'systemctl status mysql'}`
     - `確認埠號正確: ${system === 'postgresql' ? '預設 5432' : '預設 3306'}`
     - `檢查 ${options.host} 是否可達: ping ${options.host} 或 telnet ${options.host} ${options.port}`

   **ETIMEDOUT** - Firewall or slow network
   - Trigger: errCode === 'ETIMEDOUT' OR message includes 'timeout' or 'timed out'
   - Message: `連接超時 (${options.timeout || 5000}ms) — 可能是防火牆阻擋或網路延遲`
   - Hints:
     - `檢查防火牆: ${system === 'postgresql' ? '允許 TCP 5432' : '允許 TCP 3306'}`
     - `增加超時時間: 編輯 .dbcli 並新增 "timeout": 15000`
     - `確認網路連接: ping ${options.host} -c 3`

   **AUTH_FAILED** - Authentication (credentials) error
   - Trigger: message includes 'authentication' OR 'auth' OR 'password' OR 'access denied' OR 'FATAL'
   - Message: `認證失敗 — 檢查使用者名稱或密碼`
   - Hints:
     - `驗證認證: ${system === 'postgresql' ? 'psql -U ' + options.user + ' -h ' + options.host : 'mysql -u ' + options.user + ' -h ' + options.host}`
     - `檢查 pg_hba.conf (PostgreSQL) 或 user privileges (MySQL)`
     - `重新執行 dbcli init 以更新認證`

   **ENOTFOUND** - Host not found (DNS resolution failed)
   - Trigger: errCode === 'ENOTFOUND' OR message includes 'not found' OR 'getaddrinfo'
   - Message: `找不到主機: ${options.host}`
   - Hints:
     - `檢查主機名拼寫: ${options.host}`
     - `確認 DNS 可解析: nslookup ${options.host}`
     - `若使用 localhost，試試 127.0.0.1 (IPv4 vs IPv6 問題)`

   **UNKNOWN** - Fallback for unrecognized errors
   - Trigger: None of above match
   - Message: `連接失敗: ${errMsg}`
   - Hints:
     - `檢查連接參數: host=${options.host}, port=${options.port}, user=${options.user}`
     - `查看伺服器日誌: ${system === 'postgresql' ? 'postgresql.log' : 'mysql.log'}`
     - `嘗試直接用 ${system === 'postgresql' ? 'psql' : 'mysql'} 命令行工具測試`

import { test, expect } from 'bun:test'
import { mapError, ConnectionError } from 'src/adapters/error-mapper'
