**Key adapter responsibilities** (from research):
- connect(): Establish connection, validate credentials, throw ConnectionError with hints on failure
- disconnect(): Close connection safely (never throw), safe to call multiple times
- execute<T>(sql, params): Run parameterized query, return typed results, prevent SQL injection
- listTables(): Query information_schema, return TableSchema[] with row counts
- getTableSchema(tableName): Query information_schema for detailed column info
- testConnection(): Lightweight SELECT 1 probe to verify connectivity

**Bun.sql API reference** (from 03-RESEARCH.md):
- Import from 'bun:sql' module (if available in Bun 1.3.3+)
- Connection string format: postgresql://user:password@host:port/database
- Parameterization: $1, $2, $3... for PostgreSQL; ? for MySQL
- No external npm dependencies needed (native in Bun)

  database: 'postgres',

    }

    } finally {
      await adapter.disconnect()
    }
