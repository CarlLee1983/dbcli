  <name>Task 1: Create message-loader.ts with MessageLoader class</name>
  <files>src/i18n/message-loader.ts</files>
  <action>
Create MessageLoader singleton class (synchronous, 80-100 lines) with:
1. Private static instance variable for singleton pattern
2. Private messages and fallbackMessages objects (Record<string, any>)
3. Private currentLang property tracking selected language from Bun.env.DBCLI_LANG
4. Private constructor() calling loadMessages() synchronously at instantiation
5. Static getInstance() returning singleton, lazy-initializing on first call
6. Private loadLanguageFile(lang: string) method using Bun.file(path).json()
  - File path: path.join(import.meta.dir, `../../resources/lang/{lang}/messages.json`)
  - Check file exists via Bun.file().size > 0 (Bun docs pattern)
  - Throw if file missing: new Error(`Language file not found: {lang}`)
7. t(key: string): string method
  - Split key by '.' (e.g., "init.welcome" → ["init", "welcome"])
  - Walk nested objects: let value = messages, then for each part: value = value?.[part]
  - Return from currentLang if string, else try fallbackMessages
  - Last resort: return key name (e.g., "init.welcome")
8. interpolate(key: string, vars: Record<string, string | number>): string method
  - Get message via t(key)
  - For each var in vars, replace {varName} with String(value)
  - Use new RegExp(`{${varName}}`, 'g') for global replacement
9. Export singleton: export const messageLoader = MessageLoader.getInstance()
10. Export convenience functions:
    - export const t = (key: string): string => messageLoader.t(key)
    - export const t_vars = (key: string, vars: Record<string, string | number>): string => messageLoader.interpolate(key, vars)

Implementation details per RESEARCH.md Pattern 1:
- Language selection: Bun.env.DBCLI_LANG || 'en' (default English)
- Synchronous loading: Bun.file().json() is optimized for CLI startup (< 2ms)
- Fallback chain: loadLanguageFile() loads both requested language AND English fallback
- Import path: Use import.meta.dir (Bun-specific) not __dirname (Node)
- Handle unsupported language: If DBCLI_LANG=fr-FR and only en/zh-TW exist, fallback to English with no warning in Phase 12 (research suggestion deferred)

Per coding-style.md:
- Comprehensive error handling in loadLanguageFile()
- No console.log in class (errors thrown, not logged)
