<objective>
Implement incremental schema diffing algorithm to detect and report changes between the cached schema in `.dbcli` and the live database schema. This foundation enables the schema refresh command to intelligently update only schema deltas.

Purpose: Enable users to detect schema changes (new tables, dropped columns, modified types) without re-scanning entire database. Provides foundational architecture for Phase 08 export command.

Output: `SchemaDiffEngine` class that compares previous and current schema, returns detailed diff report; type definitions for diff data structures.

**Algorithm (copy from RESEARCH.md verbatim):**
1. Get current tables: `await this.adapter.listTables()` → extract names into Set
2. Get previous tables: `Object.keys(this.previousConfig.schema || {})` → extract names into Set
3. Detect table-level changes: difference between sets
4. For unchanged tables, call `this.adapter.getTableSchema(tableName)` to get column-level details
5. Compare columns by name (detect adds/removes)
6. For matching columns, use `columnChanged()` helper to detect type/nullable/default/primaryKey changes
7. Return `SchemaDiffReport` with: `tablesAdded`, `tablesRemoved`, `tablesModified` (keyed by table name)
8. Generate human-readable summary: `"${tablesAdded.length} added, ${tablesRemoved.length} removed, ${Object.keys(tablesModified).length} modified"`

**Type normalization in columnChanged():** Normalize both types to lowercase before comparison to handle database-specific formatting differences (e.g., VARCHAR vs varchar).
