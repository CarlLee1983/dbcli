Create src/core/query-executor.ts with QueryExecutor class. This handles the core logic: permission validation, query execution, result collection, error handling with suggestions.

QueryExecutor class:
```typescript
export class QueryExecutor {
  constructor(
    private adapter: DatabaseAdapter,
    private permission: Permission
  ) {}

  async execute(sql: string, options?: { autoLimit?: boolean; limitValue?: number }): Promise<QueryResult<Record<string, any>>>
}
```

Implementation steps:

1. **Execute method signature**:
   ```typescript
   async execute(
     sql: string,
     options?: {
       autoLimit?: boolean
       limitValue?: number
     }
   ): Promise<QueryResult<Record<string, any>>>
   ```

2. **Permission enforcement** (before execution):
   - Import enforcePermission from '@/core/permission-guard'
   - Call: `const classification = enforcePermission(sql, this.permission)`
   - If enforcePermission throws PermissionError, catch and re-throw with user message
   - If classification.confidence === 'LOW', log warning but do NOT block (HIGH/MEDIUM allowed)

3. **Auto-limit in query-only mode** (safety default):
   - Check: if permission === 'query-only' AND !sql.match(/LIMIT\s+\d+/i)
   - AND options.autoLimit !== false (allow override)
   - Auto-append: ` LIMIT ${options?.limitValue || 1000}`
   - Log info: "Query-only mode: auto-limiting to {limit} rows" (to stderr)
   - Reference: RESEARCH.md lines 118-130

4. **Execute query**:
   - Measure execution time: `const start = performance.now()`
   - Call: `const rows = await this.adapter.execute(sql)`
   - Calculate: `const executionTimeMs = Math.round(performance.now() - start)`
   - Handle adapter errors (connection failures already caught by adapter)

5. **Collect result metadata**:
   - Extract columnNames from rows[0] if rows exist: `Object.keys(rows[0] || {})`
   - If rows exist, infer column types from first row values (runtime type inference):
     * number → 'integer' or 'decimal'
     * string → 'varchar'
     * boolean → 'boolean'
     * Date → 'timestamp'
     * null → 'null'
   - Note: Keep simple—don't query information_schema, infer from data

6. **Build QueryResult object**:
   ```typescript
   const result: QueryResult<Record<string, any>> = {
     rows,
     rowCount: rows.length,
     columnNames,
     columnTypes: inferred,
     executionTimeMs,
     metadata: {
       statement: classification.type,
       affectedRows: rows.length // For INSERT/UPDATE/DELETE, may be 0
     }
   }
   return result
   ```

7. **Error handling with suggestions**:
   ```typescript
   try {
     // execute query
   } catch (error) {
     if (error instanceof PermissionError) {
       // Re-throw permission errors as-is
       throw error
     }

     // For database errors, try to suggest missing table
     const errorMessage = (error as Error).message
     if (errorMessage.includes('does not exist') || errorMessage.includes('not found')) {
       const {suggestions, tables} = await suggestTableName(errorMessage, this.adapter)
       const enhancedError = new Error(
         `${errorMessage}\n` +
         (suggestions.length > 0
           ? `Did you mean: ${suggestions.join(', ')}?`
           : `Available tables: ${tables.slice(0, 5).join(', ')}...`)
       )
       throw enhancedError
     }

     // Otherwise throw original error
     throw error
   }
   ```

8. **Type checking**:
   - Ensure QueryResult<Record<string, any>> is properly imported from src/types/query
   - Ensure DatabaseAdapter is properly typed
   - Ensure Permission type is properly imported

Reference RESEARCH.md lines 46-58 for flow diagram and lines 96-116 for error handling strategy.

     // ... execute and format

       console.error(`   Required: ${error.requiredPermission} mode`)
       console.error(`   Message: ${error.message}`)
       process.exit(1)

     if (error instanceof ConnectionError) {
       console.error(`❌ Database Connection Failed`)
       console.error(`   ${error.message}`)
       process.exit(1)
     }

     // Other errors (missing table, syntax, etc.)
     console.error(`❌ Query Error`)
     console.error(`   ${error.message}`)
     process.exit(1)
   }
